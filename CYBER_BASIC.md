# 網路基礎概念
## IP 地址 (IP Address)
- 定義：IP 地址是分配給網路設備的唯一類似地址的一長串數字，用於在網路中進行通信。IP 地址有兩種類型：IPv4 和 IPv6。
- IPv4：由 32 位元組成，通常表示為 4 個十進位數字（如 192.168.1.1）。
- IPv6：由 128 位元組成，提供更大的地址空間，通常表示為 8 組 16 進位數字（如 2001:0db8:85a3:0000:0000:8a2e:0370:7334）。
### QUESTIONS
- **欸....所以早期是IPV4現在是IPV6嗎?既然IPV6比IPV4更加廣闊,所以IPV6是現在被廣泛使用的吧?**
- IPv6 的確比 IPv4 提供了更大的地址空間和技術優勢，但由於各種原因，IPv4 仍然在全球範圍內被廣泛使用。
- IPv6 的採用正在逐步增加，但目前大部分網絡仍然依賴於雙棧技術，以兼容 IPv4 和 IPv6。
- 隨著時間的推移和互聯網的進一步發展，IPv6 的使用將會越來越普遍，但這個過渡仍需要時間。
## 子網掩碼 (Subnet Mask)
- 定義：子網掩碼用來將 IP 地址分為網路部分和主機部分。子網掩碼與 IP 地址一起使用來確定兩個主機是否位於同一子網中。
- 例如，IP 地址 192.168.1.10 與子網掩碼 255.255.255.0 結合後，表示該主機位於 192.168.1.x 網段中。
### QUESTIONS
- **是要怎麼結合?可以說明的更具體一點嗎?有沒有甚麼規律?**
![image](https://github.com/user-attachments/assets/5e0eb7e2-ca53-436a-af6b-792ad45096e8)
- 網路部分是前 25 位（即前三個八位元組和第四個八位元組中的前一位），這代表該網段內的所有 IP 地址都共享這部分。
- 主機部分是剩下的 7 位（第四個八位元組的剩餘位），這部分可以變化，表示這個網段內的不同主機地址。
- 理論上最多可以有 31 個 1 位（/31），通常用於點對點連接,最少可以有 1 個 1 位（/1），這會產生一個非常大的網段。
- /32 則表示單一 IP 地址，沒有主機部分。
## 預設閘道 (Default Gateway)
- 定義：預設閘道是指網路設備在與不同網段的設備通信時所使用的路由器 IP 地址。當設備需要與不同子網或網段中的設備通信時，數據包會首先被發送到預設閘道。
### QUESTIONS
- **數據包是封包的意思嗎?所以數據包會線從本地端的設備發送到預設閘道然後在向目標閘道發送過去嗎?**
- 是的，數據包（Packet）是網絡通信中傳輸數據的一種基本單位，可以理解為“封包”。當設備在網絡中進行通信時，數據通常會被分割成許多數據包，每個數據包包含了數據以及其目的地的地址等信息。
- 預設閘道是本地網絡設備發送數據包到其他網段或子網的第一跳，類似於一個“中轉站”。
- 當數據包需要跨越不同的網段時，它會先被發送到預設閘道，預設閘道會根據目標地址決定如何將數據包轉發到目標網絡。
## DNS (域名系統, Domain Name System)
- 定義：DNS 是將人類可讀的域名（如 www.example.com）解析為 IP 地址（如 192.0.2.1）的系統，便於在網路中定位和訪問網站或服務。
- 功能：它充當網路中的“電話簿”，當你在瀏覽器中輸入域名時，DNS 會將其轉換為相應的 IP 地址。
### QUESTIONS
- **所以DNS就是將網路的URL轉換為IP地址的一個通訊系統嗎?**
- 是的，你理解得非常正確。DNS（Domain Name System，域名系統）是一個將人類可讀的域名（如 www.example.com）轉換為機器可讀的 IP 地址（如 192.0.2.1）的系統。
- DNS 的層級結構：
- 根域名服務器：位於 DNS 層級結構的最頂端，負責指向 TLD（頂級域名）伺服器。全球只有 13 個主要的根伺服器，但它們有數百個鏡像，分布在世界各地。
- 頂級域名 (TLD) 伺服器：管理如 .com、.org、.net、國家代碼（如 .cn、.uk）等頂級域名，負責指向次級域名伺服器。
- 權威域名伺服器：管理特定域名的 DNS 記錄，提供該域名最終的 IP 地址。例如，example.com 的權威 DNS 伺服器就會返回 www.example.com 的 IP 地址。
### QUESTIONS
- **DNS伺服器又是甚麼?主要分布在哪裡?**
- DNS 伺服器的作用
- 本地 DNS 伺服器：通常由 ISP（網絡服務供應商）或企業內部設置，用於處理客戶端發起的 DNS 查詢。
- 權威 DNS 伺服器：負責提供特定域名的最終 IP 地址，這些伺服器存儲了該域名的 DNS 記錄，包括 A 記錄（IPv4 地址）、AAAA 記錄（IPv6 地址）、MX 記錄（郵件交換器）等。
- 根 DNS 伺服器：全球有 13 個主要的根 DNS 伺服器（用字母 A 到 M 命名），這些伺服器及其分佈的鏡像負責整個 DNS 系統的運行。根伺服器通常由非營利組織或政府機構管理，如 ICANN、Verisign 等。
## DHCP (動態主機配置協議, Dynamic Host Configuration Protocol)
- 定義：DHCP 是一種網路協議，用於自動分配 IP 地址和其他網路配置（如子網掩碼、預設閘道、DNS 伺服器）給網路設備。
- 功能：DHCP 減少了手動配置每個網路設備的 IP 地址的需求，並確保 IP 地址不會重複或衝突。
### QUESTIONS
- **是每個路由器都有DHCP的功能嗎?**
- 是的，大多數現代路由器都具有 DHCP 功能，這使得它們能夠自動分配 IP 地址給連接到網絡的設備。這對於家庭網絡和小型辦公室網絡尤其有用。雖然大多數路由器默認啟用了 DHCP 服務，但你也可以手動進行配置和調整。
## MAC 地址 (媒體訪問控制地址, Media Access Control Address)
- 定義：MAC 地址是分配給網絡設備的硬體地址，通常由網卡製造商在出廠時設定。它由 48 位二進位數組成，通常表示為 12 個 16 進位數字（如 00:1A:2B:3C:4D:5E）。
- 功能：MAC 地址用於網路層以下（如以太網）進行設備識別和通信。
### QUESTIONS
- **所以網路層以下的設備是別都是使用MAC地址....網路層以上使用的是IP地址嗎?**
- 你理解得非常正確。MAC 地址和 IP 地址在網絡通信中的確分別在不同的層次上發揮作用。
- MAC 地址主要在數據鏈路層（Data Link Layer）進行通信，用於標識和傳輸數據到同一個局域網內的設備。
- IP 地址主要在網絡層（Network Layer）進行通信，用於定位和路由數據包(封包)，使其能夠跨越不同的網絡到達目的地。
- MAC 地址與 IP 地址的協作：IP 地址負責確定數據包的路徑，而 MAC 地址則負責實際的數據傳輸，特別是在同一局域網內。
## OSI 模型 (開放式系統互聯模型, Open Systems Interconnection Model)
- 定義：OSI 模型是一個概念框架，用來描述網路通信過程中的不同層級架構。它由 7 層組成，從最高層到最低層分別是：
- 應用層 (Application Layer)：為應用程序提供網路服務，如 HTTP、FTP、SMTP。
- 表示層 (Presentation Layer)：處理數據的編碼和加密，確保不同系統之間的數據格式一致。
- 會話層 (Session Layer)：管理通信會話的建立、維護和終止。
- 傳輸層 (Transport Layer)：提供端到端的數據傳輸和錯誤檢測，如 TCP/UDP。
- 網路層 (Network Layer)：負責邏輯地址尋址和路由選擇，如 IP
- 數據鏈路層 (Data Link Layer)：負責介質訪問控制和物理地址尋址，如 Ethernet。
- 物理層 (Physical Layer)：傳輸原始比特流的介質，如電纜、無線電波。
### QUESTIONS
- **那....通常電信業者的主要的服務範圍是physical layer到network layer嗎?**
- 是的，電信業者（ISP， Internet Service Providers）的服務範圍主要集中在 OSI 模型的物理層 (Physical Layer) 到網路層 (Network Layer)。
- 電信業者的核心服務範圍主要集中在 OSI 模型的物理層到網路層，涵蓋了物理連接、數據鏈路管理、和網路層的 IP 路由和尋址。
## LAN 和 WAN (局域網與廣域網)
- 局域網 (Local Area Network, LAN)：
- 定義：LAN 是指在有限地理區域（如辦公室、家庭）內的網路，通常由幾台到數百台設備組成。
- 特性：LAN 提供高帶寬和低延遲的網路環境，常見技術包括 Ethernet 和 Wi-Fi。
- 廣域網 (Wide Area Network, WAN)：
- 定義：WAN 是指跨越大範圍地理區域（如城市、國家或大洲）的網路，由多個 LAN 連接組成。
- 特性：WAN 通常使用路由器和公共或專用的傳輸鏈路（如 MPLS、VPN）進行連接，網路延遲較高但覆蓋範圍廣。
### QUESTIONS
- **所以從路由器透過網址到外面的伺服器等等都是WAN廣域網嗎?,從路由器到本地端的通訊設備是LAN局部網嗎?**
- 是的，你的理解是正確的。在日常使用中，LAN 負責本地網絡內的通信，而 WAN 則負責本地網絡與外部世界的通信。這兩者的協同工作讓我們能夠在全球範圍內方便地訪問信息和服務。
- **我看一些伺服器有接兩條網路線,一條是接到WAN,一條是接到LAN,為甚麼要這樣接?一般的電腦主機可以灌系統做成一台伺服器嗎?**
- 伺服器上接兩條網路線，一條接到 WAN（廣域網），一條接到 LAN（局域網），通常是為了實現更好的網絡管理、安全性、以及網絡流量的隔離。
- 為什麼伺服器會同時連接到 WAN 和 LAN？多網卡伺服器：伺服器安裝多個網絡接口卡 (NIC)，允許它同時連接到不同的網絡。這通常有以下幾個原因：網絡隔離與安全性,負載平衡與冗餘,不同網段的通信,特定服務的隔離
- **一般的電腦主機可以灌系統做成一台伺服器嗎?**
- 可以!
- 硬件要求
- 處理器：至少需要一個多核處理器，根據所需的負載情況選擇更高性能的處理器。
- 內存：建議至少 8 GB 內存，伺服器負載越大，所需內存也越多。
- 儲存：需要大容量硬碟，最好使用 RAID 配置以提高數據可靠性。
- 網卡：至少需要一個網卡，如果需要多網段或隔離，可以考慮安裝多個網卡。
## TCP/IP 協議
- 定義：TCP/IP 是一組網路協議，定義了在互聯網上如何傳輸數據。它由兩個主要協議組成：
- TCP (傳輸控制協議, Transmission Control Protocol)：提供可靠的數據傳輸服務，通過確認和重傳機制確保數據完整性。
- IP (網際協議, Internet Protocol)：負責數據包的尋址和路由，確保數據能夠在網路上找到正確的目的地。

## 防火牆 (Firewall)
- 定義：防火牆是一種網路安全設備，用於監控和控制進出網路流量，根據預定的安全規則來允許或阻止數據包的傳輸。
- 功能：防火牆可以保護網路免受未經授權的訪問和攻擊。
### QUESTIONS
- **在不同的地方會有不一樣功能的防火牆嗎?有大概分為哪幾種種類嗎?**
- 防火牆可以根據部署方式、功能和技術分類，如網絡層防火牆、應用層防火牆、專門用途防火牆等。
- 不同類型的防火牆適用於不同的環境和需求，如保護內部網絡、Web 應用、雲基礎設施等。
- 硬件、軟件和虛擬防火牆提供了靈活的部署選項，適合不同的網絡架構和安全需求。
- 網絡層防火牆:
- 包過濾防火牆 (Packet-Filtering Firewall)
- 狀態檢測防火牆 (Stateful Inspection Firewall)
- 應用層防火牆:
- 代理防火牆
- 下一代防火牆 (Next-Generation Firewall, NGFW)：功能：結合了傳統防火牆、入侵檢測和預防系統 (IDS/IPS)、應用層過濾等功能。NGFW 可以深入檢查數據包的內容，識別和阻止特定應用或威脅。
- 應用場景：用於保護複雜的企業網絡，防止先進的網絡威脅。
- 專門用途防火牆:
- Web 應用防火牆
- 雲防火牆
- 無線防火牆
